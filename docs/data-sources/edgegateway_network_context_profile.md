# cloudavenue_edgegateway_network_context_profile (Data Source)

The `cloudavenue_edgegateway_network_context_profile` data source allows you to retrieve information about a Network Context Profile (Layer 7) available on an Edge Gateway. Use this to reference SYSTEM or PROVIDER profiles by name in firewall rules.

Network Context Profiles are available in three scopes:
- **SYSTEM** — predefined by VMware/NSX-T (e.g. `SSL`, `CIFS`, `HTTP`, `DNS`, `SSH`). Read-only.
- **PROVIDER** — defined by Orange at platform level. Read-only.
- **TENANT** — created by the customer via `cloudavenue_edgegateway_network_context_profile`. Fully manageable.

Use this data source to reference a SYSTEM or PROVIDER profile by name in a Firewall Rule via `network_context_profile_ids`.


## Example Usage

```terraform
data "cloudavenue_edgegateway_network_context_profile" "example" {
  edge_gateway_name = cloudavenue_edgegateway.example.name
  name              = "SSL"
}
```

## Advanced Usage

Use the profile ID in a Firewall Rule:

```terraform
data "cloudavenue_edgegateway_network_context_profile" "ssl" {
  edge_gateway_name = cloudavenue_edgegateway.example.name
  name              = "SSL"
}

resource "cloudavenue_edgegateway_firewall" "example" {
  edge_gateway_name = cloudavenue_edgegateway.example.name
  rules = [
    {
      name        = "allow outbound SSL"
      action      = "ALLOW"
      direction   = "OUT"
      ip_protocol = "IPV4"

      network_context_profile_ids = [data.cloudavenue_edgegateway_network_context_profile.ssl.id]
    }
  ]
}
```
